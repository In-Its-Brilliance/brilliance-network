Тест консистентности доставки unreliable сообщений через сетевой слой.

Клиент шлёт `PlayerMove` (Unreliable) каждые ~16мс (64 TPS).
Сервер принимает их и отправляет `EntityMove` (Unreliable) обратно клиенту.

Оба направления проверяются:

- **Сервер** — считает сколько `PlayerMove` пришло за каждый тик. Если сетевой слой доставляет сообщения равномерно, на каждый тик должно приходить 0 или 1 сообщение. Если приходит >1 за тик — значит сетевой слой батчит. Также проверяет порядок (sequence) и потери.

- **Клиент** — считает сколько `EntityMove` пришло за каждый тик обратно от сервера. Проверяет батчинг в обратном направлении (сервер → клиент) и потери.

```shell
# Терминал 1 - сервер
cargo run -p network-consistency -- -t server

# Терминал 2 - клиент
cargo run -p network-consistency -- -t client

# С параметрами
cargo run -p network-consistency -- -t server --ip=127.0.0.1:25570 -d 10
cargo run -p network-consistency -- -t client --ip=127.0.0.1:25570 -d 10
```
